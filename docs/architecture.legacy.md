# 顶层架构

## 总体思想

E.coil 的结构可以理解为：

```text
生物数据库
  -> 细胞知识图谱
  -> 分子 Agent
  -> 多轮状态演化
  -> 全量历史
  -> ReportAgent 解释
```

它参考 MiroFish 的“种子材料建图 -> 构建世界 -> Agent 回合互动 -> 记录历史 -> 生成报告”流程，但把社会世界替换成细胞内部世界。

## 与 MiroFish 的关键差异

MiroFish 的 Agent 之间主要通过语言互动，涌现来自语义理解和群体叙事演化。

E.coil 的 Agent 之间主要通过标准状态传导，涌现来自知识图谱上的级联响应。

因此，LLM 在 E.coil 中不是“对话引擎”，而是“状态决策引擎”：

```text
读取：当前 Agent 的功能注释、状态、邻居变化、候选规则
判断：邻居变化是否影响当前 Agent
输出：受规则约束的 action
落地：ActionInterpreter 归类为标准状态
```

这个设计比自由对话更结构化，也更容易被规则注册表约束。

## 系统模块

### 1. 扰动解析 Agent

输入用户自然语言，输出结构化扰动。

例子：

```text
用户：敲除 lacI，并加入乳糖
输出：
  - set gene.lacI = knocked_out
  - set molecule.lactose = high
```

外源药物或诱导物不作为系统实体预先加入图谱。它们在实验开始时以自然语言进入扰动解析层。

推荐采用“图谱预筛 + LLM 精判”：

```text
用户输入扰动
  -> 关键词/同义词检索
  -> 在知识图谱中找到候选实体和候选通路
  -> 只把几十个候选实体的注释喂给 LLM
  -> LLM 判断哪些原生实体直接受影响
  -> 输出本次实验的初始状态变更
```

例子：

```text
加入青霉素
  -> 图谱预筛命中 PBP、细胞壁合成、肽聚糖相关实体
  -> LLM 精判直接受影响实体
  -> 输出 protein.PBP_x = inhibited 等初始状态变化
```

### 2. 知识图谱层

知识图谱决定 Agent 之间能否交互。

图谱节点：

- 基因
- 蛋白
- 小分子
- 复合体
- 通路
- 表型标签

图谱边：

- encodes
- regulates
- activates
- inhibits
- catalyzes
- transports
- binds
- forms_complex
- participates_in_pathway

### 3. Agent 层

每个分子种类是一个常驻 Agent。

Agent 只知道：

- 自己的功能注释。
- 自己所在通路。
- 自己的图谱邻居。
- 自己和邻居的历史状态。

Agent 不知道全细胞所有信息，避免模型自由发挥。

### 4. 仿真循环

每一轮中，Agent 根据邻居状态更新自己的离散状态，并记录理由。

LLM 的作用不是“发明生物规则”，而是根据已有关系、功能注释和邻居状态解释并选择合理的状态变化。

传导过程中，不是所有邻居都会响应。一个 Agent 看到邻居状态变化后，需要判断：

```text
我的邻居发生了变化。
根据我的功能注释和我与它的图谱关系，
这个变化是否会影响我？
如果会，我应该进入什么离散状态？
如果不会，我保持当前状态。
```

这正是 LLM 的核心价值：用语义理解功能注释，而不是把每一种传导关系都硬编码成规则。

### 5. 报告与响应模式对照

推理结束后的主要产物是 ReportAgent 的全局生物学报告。所谓“表型”在第一版里不要做成仿真的硬输出，更适合放在报告阶段作为可选的已知响应模式对照。

例子：

- `lac_operon_derepression_like`
- `aminoglycoside_translation_response`
- `sos_response_like`
- `oxidative_stress_response_like`
- `beta_lactam_cell_wall_response`

这些模式不会参与状态传导，只用于报告里提示“最终状态组合像哪类已知响应”。

### 6. ReportAgent

ReportAgent 在仿真结束后读取完整历史，回答：

- 扰动从哪里开始？
- 沿哪条通路或调控网络传导？
- 哪些 Agent 是意外响应或涌现候选？
- 最终系统稳定在什么状态？
- 这个状态组合是否接近某类已知突变体或处理响应？
- 某个 Agent 当时为什么改变状态？
- 如果去掉某个扰动，可能会怎样？

MiroFish 的 ReportAgent 复盘的是语义内容和舆论走向；E.coil 的 ReportAgent 复盘的是状态变化轨迹、通路级联和可选的响应模式匹配。

E.coil 的潜在优势是可以把仿真结果和真实生物知识库比较：

```text
这次扰动产生的状态组合与 ΔlacI-like response pattern 相符。
第 7 轮 ArcA 的意外激活不属于当前已知响应模式，可能是一个待验证假说。
```

因此 ReportAgent 不只是复盘工具，也可以成为生物学假说生成器。

## 最重要的边界

LLM 可以做：

- 状态解释。
- 候选状态变化选择。
- 因果链总结。
- Agent 重新实例化对话。

LLM 不可以做：

- 编造新的通路。
- 编造新的蛋白功能。
- 编造新的调控边。
- 编造新的药物靶点。
- 直接修改知识图谱。
